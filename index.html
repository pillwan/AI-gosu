🔍 AI 이전 시대 — 키워드 검색

"구글/네이버에 키워드 입력 → 링크 10개 중 골라 읽기가 전부였다"

정보는 있지만 찾는 건 사람의 몫 — 클릭, 비교, 판단 전부 수동

핵심 한계: 질문 맥락 모름, 후속 질문 불가, 정보 조합은 사람이 직접
SLIDE 06

💬 1단계 — 대화형 검색 (2022~)

"기계가 사람의 의도를 이해하기 시작한, AI 역사상 가장 파격적인 전환점"

전환점: ChatGPT 공개 (2022.11.30) — 모든 것이 달라진 날

변화의 본질: AI가 처음으로 사람의 "의도"를 이해하게 됨

❌ 이전 — 키워드 검색

키워드 입력 → 링크 10개 나열

사람이 일일이 클릭 · 비교 · 판단

기계는 사람의 의도를 모름

✅ 이후 — 대화형 AI

자연어로 질문 → 맥락 이해한 답변

후속 질문 · 재질문 가능

AI가 의도를 이해하고 응답

💡 왜 파격적인가: 검색은 원래 사람이 기계 언어(키워드)에 맞춰 주던 일이었습니다. 이제 기계가 사람의 언어를 이해합니다. 인간-기계 관계의 근본이 뒤집힌 순간입니다.
SLIDE 07

⚡ 2단계 — 생산성 도구 (2023~)

"찾아주는 AI에서 만들어주는 AI로, 업무 시간 절반 단축"

전환점: GPT-4, Claude, 기업용 도입 본격화

변화의 본질: 이해한 내용을 결과물로 생산하기 시작

1
문서 작성
이메일, 보고서, 기획안 — 가장 많이 쓰이는 용도
2
코딩 / 개발
IT 업계에서 1순위 — 코드 생성, 리뷰, 디버깅
3
번역 · 요약 · 분석
맥락 이해 기반의 자연스러운 결과물
4
이미지 · 영상 생성
DALL-E, Midjourney, Sora — 창작 영역까지 확장
🎯 핵심 변화: "찾아줘"에서 "만들어줘"로. AI가 결과물을 직접 생산하며 업무 생산성이 구조적으로 바뀝니다.
SLIDE 08

🤖 3단계 — 에이전트 AI / Agentic AI (2025~)

"AI가 답을 주는 것을 넘어, 사람을 대신해 직접 실행하는 시대"

전환점: Claude Code, GPT Operator, MCP 표준화

Agent의 뜻: 대리 · 대행 — 사람을 대신해 일을 처리하는 주체

에이전트 AI
Agent = 대리 · 대행
사람을 대신해 실행
실행 AI
답변이 아닌 행동
파일·명령·외부 연동
자율 · 능동실행 AI
사람 개입 없이
스스로 판단 · 수행
💡 핵심: 사람은 지시하고 검토, AI가 실행합니다. "AI가 답을 준다"에서 "AI가 일을 한다"로의 전환 — 위임의 시대가 열립니다.
SLIDE 09

🔑 왜 에이전트 AI는 '지금' 가능한가

"세 가지 기반 기술이 동시에 성숙해지며, 위임이 현실이 되었다"
1
긴 컨텍스트 (200K~1M 토큰)
프로젝트 전체를 한 번에 이해 → 긴 작업을 맡길 수 있게 됨
2
Tool Use (도구 사용)
AI가 파일을 읽고·쓰고, 명령어를 실행하고, 외부 API를 호출
3
MCP 표준화
GitHub · Gmail · DB 등 외부 시스템을 통일된 규약으로 연결
📌 결론: 에이전트 AI는 단순 유행이 아닙니다. 기반 기술이 성숙했고, 이제 사람이 AI에게 일을 통째로 맡길 수 있게 된 구조적 변화입니다.

🔄 AI가 바꾸는 업무 방식

"AI는 사람을 대체하는 것이 아니라 생산성을 10배로 높이는 도구"

❌ As-is (기존)

  • 수동 문서 작성 (2~3시간)
  • 코드 리뷰 대기 (반나절~1일)
  • 매뉴얼 검색 후 직접 번역
  • 데이터 분석 스크립트 직접 작성
  • 회의록 수기 정리

✅ To-be (AI 활용)

  • AI 초안 + 사람 검토 (30분)
  • AI 코드 리뷰 즉시 피드백
  • AI 번역 + 맥락 보정 (5분)
  • 자연어로 분석 요청 → 즉시 결과
  • AI 자동 요약 + 액션 아이템 추출

⚙️ Hooks — 이벤트 자동 트리거

"파일 저장 후 자동 검사, Stop 후 알림 — 정해진 순간 자동 실행"

이벤트 종류

이벤트발생 시점활용 예
PreToolUse도구 실행 전위험 명령 차단, 입력 수정
PostToolUse도구 실행 후자동 lint·포맷팅, 검증
Stop세션 종료 시데스크톱 알림, 결과 저장
UserPromptSubmit유저 입력 제출 시입력 가공, 컨텍스트 추가

설정 위치 + 예시

사용자 전체: ~/.claude/settings.json · 프로젝트별: .claude/settings.json (Git 커밋 가능)

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|MultiEdit|Write",
        "hooks": [
          { "type": "command", "command": "npm run lint" }
        ]
      }
    ]
  }
}

활용 사례: 1) 파일 저장 후 자동 lint · 2) Stop 시 데스크톱 알림 · 3) rm -rf 차단 · 4) 민감 키워드 입력 차단

🎯 핵심: Hook은 Skill·MCP와 다릅니다. "언제" 무엇이 일어나야 하는가의 답.
